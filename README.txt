Гайд

Квест командный, состоит из двух частей: онлайн(задания выдаются через бота, ответы тоже) и оффлайн(
задания выдаются на КП, команде нужно только сделать "чек ин" и "чек аут" на каждой КП).

Настройка бота под Квест

Онлайн-часть:
0. В config.py: MODE = ONLINE - установить конфигурацию игры на Онлайн-часть
1. Если csv-таблицы еще не заполнены: загрузить все файлы в лс бота, бот выведет их file_id.
    Заполнить с ними file.csv.
2. Убедиться, что в директории data лежат on_point.csv, off_point.csv, file.csv, reaction.csv, и они
    верно заполнены(данными по сценарию квеста). В on_point и off_point поле name должно быть равно LastPoint(настраивается в config)
3. Необходимо обозначить админов, для этого в config.py надо заполнить список ADMIN_USER_IDS(
    telegram user_id пользователей, которым должны быть доступны все команды).
4. На локальном компьютере - запустить бота(main.py), на сервере(heroku) - бот запустится сам.
    Бот не сможет работать одновременно и там, и там.
5. Использовать команду /fill - заполнить базу данных из csv файлов.
6. Создать групповые чаты в telegram, добавить туда бота.
7. В BotFather разрешить боту читать чужие сообщения(/setprivacy)
8. В чате зарегистрировать команды(привязка к данному чату):
	/reg название кол-во_участников секция
	Секцию указать 1 или 2 - это круг на Оффлайн части(см. сценарий)
9. Ввести /pinned и закрепить отправленное ботом сообщение
10. Удалить все сообщения кроме pinned
11. Косметика беседы - название, ава и тд
12. Добавить участников команды(зарегистрировавшихся через форму)
13. Для ввода ответа надо ввести команду /try ответ. Попыток обычно несколько.
14. Для того, чтобы пропустить задание, ввести /skip
15. После прохождения онлайн-части текущей командой, доступ к игре будет у нее закрыт.

Оффлайн:

1. В config.py поменять: MODE = OFFLINE_MODE - конфигурация игры на Оффлайн-часть
2. Ввести /flush в лс с ботом - это сбросит статус игры команды на "Ожидание начала Оффлайн-части"
3. Команда должна ввести /kill в чате(после того, как организаторы об этом сообщат команде вживую)
4. Появившись на КП, команда должна сразу ввести команду /in код_начала для начала прохождени КП,
    после чего пойдет отсчет времени. Команда может пройти КП быстро, средне, медленно, либо не пройти вообще.
5. Когда команда выполнит задание от КПшника, она должна ввести /checkout код_конца для окончания
    прохождения КП. Оба кода находятся на КП в реальности.
6. После прохождения оффлайн-части текущей командой, доступ к игре будет у нее закрыт.
